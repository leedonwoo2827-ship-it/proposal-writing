# AI 제안서 작성 스킬 (Skill 3)

## 개요

이 스킬은 **Skill 1(RFP 뼈대)** + **Skill 2(회사 데이터)**를 결합하여 제안서 초안을 작성하고,
**마크다운 형식(proposal-body.md)**으로 출력합니다.

---

## ⚠️ 핵심 원칙: 출력 형식

> **Skill 3의 최종 결과물은 반드시 `proposal-body.md` 파일로 저장되어야 한다.**

이 마크다운 파일은 hwpx-writer MCP의 `md_to_hwpx` 도구로 전달되어 HWPX 문서로 변환된다.

---

## 주요 기능

### 1. 데이터 기반 컨텐츠 합성

**입력:**
- **Skill 1**: RFP에서 추출한 Backbone (섹션 구조, depth 1~2)
- **Skill 2**: Reference 제안서 + Raw 데이터 (수치, 팩트)

**처리:**
- Reference의 논리 구조와 톤 학습
- Raw 데이터의 핵심 키워드/수치 추출
- 섹션별로 적절한 데이터 매칭

**출력:**
- 마크다운 파일 (`proposal-body.md`)

---

### 2. 분량 최적화 엔진

설정된 프리셋(5만~20만 자)에 맞춰:
- 중요 섹션 → Reference의 상세한 논리 전개
- 일반 섹션 → 요약된 사실 중심 서술

---

### 3. 근거 기반 컬러링 (인라인 마커)

출처 텍스트를 **문장 안에 자연스럽게 녹여** 넣고, 구절 단위로 색상 마커를 표시한다:
- \`{{red:구절}}\`: Raw 데이터 기반 텍스트
- \`{{green:구절}}\`: Reference/RFP 기반 텍스트
- 마커 없음: AI 생성 텍스트 (기본 검정)

마크다운 예시:
\`\`\`markdown
본 훈련과정은 {{red:20010105 빅데이터분석 세분류}}를 적용하며, {{green:산업 현장의 실무 역량 강화}}를 핵심 목표로 설계되었다.
\`\`\`

> **핵심 원칙 — 융합 작성**: 출처 텍스트를 별도 블록으로 나열하지 않는다.
> 반드시 AI가 작성한 문장 흐름 안에 출처 구절을 자연스럽게 섞어 넣는다.

---

## 마크다운 작성 규칙

### 헤딩 구조

마크다운 헤딩 `#`~`######` 6단계로 작성하면, hwpx-writer가 `proposal-styles.json` 기준으로 기호/폰트/여백을 자동 적용합니다.

- `#` → 제목 (`# 1.`, `# 1.1`, `# 1.1.1` 번호 체계 사용)
- `##` → 기호 없는 일반 본문
- `###` → 본문 항목 (기호 자동 적용)
- `####` → 세부 항목 (기호 자동 적용)
- `#####` → 보충 설명 (기호 자동 적용)
- `######` → 참고/주석 (기호 자동 적용)

> **기호를 직접 입력하지 마세요!** 헤딩 레벨에 따라 hwpx-writer가 자동으로 기호를 추가합니다.
> 기호/폰트/여백 세부사항은 hwpx-writer의 `proposal-styles.json`에서 일괄 관리됩니다.
>
> **⚠️ 위 헤딩 구조 설명은 AI 작성 규칙이며, 제안서 본문(proposal-body.md)에 레벨 가이드 표나 설명을 포함하지 마세요.**

### 볼드(\`**\`) 처리

> ⚠️ **\`**볼드**\` 마크다운은 사용하지 않는다.**
> 강조가 필요하면 색상 마커(\`{{red:}}\`, \`{{green:}}\`) 또는 헤딩 레벨을 활용한다.
> 표 안에서도 \`**\`는 절대 사용하지 않는다.

### 작성 예시

\`\`\`markdown
# 파이썬과 머신러닝으로 완성하는 경영데이터 분석

# 1. 훈련과정 개요

## 과정의 목표와 배경

본 훈련과정은 {{red:AI 기초(10차시), AI 심화(10차시)}} 등 총 5개 과정을 통합·재구성하였습니다.

### 데이터 수집 및 정제

SQL 활용 데이터 추출, 데이터 정제 및 전처리 과정을 학습합니다.

#### 주요 활용 도구

Python pandas, NumPy 라이브러리를 활용합니다.

##### 실습 환경

Jupyter Notebook 기반 실습 환경을 제공합니다.

###### 참고 사항

## 주의사항

1. ✅ **DO**: 마크다운 헤딩 \`#\`~\`######\` 사용 → 기호/스타일 자동 적용
2. ❌ **DON'T**: 기호를 직접 입력 (예: \`□ 항목\`, \`○ 항목\`)
3. ✅ **DO**: 색상 마커 \`{{red:}}\`, \`{{green:}}\` 사용
4. ❌ **DON'T**: \`**볼드**\` 마크다운 사용
5. ✅ **DO**: 표는 마크다운 테이블 문법 사용
6. ❌ **DON'T**: \`-\` 마크다운 리스트 문법 사용
7. ❌ **DON'T**: 헤딩 구조/레벨 가이드 표를 제안서 본문에 포함 (이 정보는 AI 작성 규칙일 뿐, 출력물에 넣지 않는다)
